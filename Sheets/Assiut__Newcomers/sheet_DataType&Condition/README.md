## 🎯 About

This repository documents a deliberate, long-term journey into AI and software engineering — built from first principles, not shortcuts.

Every phase is studied from **authoritative textbooks**, summarized in personal **Notion notes**, and implemented through **mini-projects** and **end-of-chapter projects**. The goal is not just to make things work — it's to understand *why* they work.

**Ultimate goal:** Grow into a highly capable AI engineer and researcher, and contribute to world-class technology systems.

---

## 🧠 Learning Approach

The same method is applied consistently across every phase and every book:

| Step | What Happens |
|------|-------------|
| 📖 **Read** | Study from the textbook deeply, not superficially |
| 💡 **Understand** | Focus on the *why* behind every concept |
| 📝 **Summarize** | Build structured Notion notes with intuition, examples, and trade-offs |
| 🤖 **AI-Assisted** | Use AI to deepen understanding, clarify doubts, and explore edge cases |
| 🔨 **Mini-Projects** | Implement a small project after every major section |
| 🚀 **Chapter Project** | Build a larger project at the end of every chapter |
| 🔁 **Repeat** | Move to the next concept with a solid foundation in place |

## 🗺️ Roadmap

---

### 🐍 Phase 1 — Python

> *Building a deep, professional-grade understanding of Python*

| Book | Focus |
|------|-------|
| 📘 **Automate the Boring Stuff with Python** — Al Sweigart | Practical Python for real-world automation |
| 📗 **Python Crash Course** — Eric Matthes | Core syntax, projects, and problem-solving |
| 📙 **Fluent Python** — Luciano Ramalho | Idiomatic Python, internals, and best practices |
| 📕 **Effective Python** — Brett Slatkin | 90 specific ways to write better Python |
| 📓 **Python Official Documentation** | Reference-level mastery of the language |

Each book is studied cover-to-cover. Every section → mini-project. Every chapter → a larger project.

---

### 📐 Phase 2 — Mathematics

> *Building the mathematical backbone required for ML and AI*

| Book | Topic |
|------|-------|
| 📘 **Calculus** → Principles of Mathematical Analysis — Walter Rudin | Real analysis, limits, differentiation, integration |
| 📗 **Linear Algebra Done Right (LADR 4e)** — Sheldon Axler | Vector spaces, linear maps, eigenvalues, spectral theorem |
| 📙 **All of Statistics** — Larry Wasserman | Probability theory, inference, and statistical modeling |
| 📕 **Mathematics for Machine Learning** — Deisenroth et al. | Bridging pure math to ML applications |

---

### ⚙️ Phase 3 — Algorithms

> *Mastering algorithmic thinking and problem-solving from first principles*

| Book | Focus |
|------|-------|
| 📘 **Introduction to Algorithms (CLRS)** — Cormen et al. | The definitive algorithms reference |
| 📗 **Algorithms 4th Edition** — Sedgewick & Wayne | Practical implementations and analysis |

**LeetCode** is practiced in parallel throughout this phase — applying every data structure and algorithm concept directly to problems.

---

### 🤖 Phase 4 — Machine Learning

> *Understanding ML from mathematical foundations to practical systems*

| Book | Focus |
|------|-------|
| 📘 **Hands-On Machine Learning** — Aurélien Géron | End-to-end ML with Scikit-Learn, Keras & TensorFlow |
| 📗 **Pattern Recognition and Machine Learning** — Bishop | The rigorous probabilistic and Bayesian treatment of ML |

---

### 🔥 Phase 5 — Deep Learning

> *Building and understanding modern deep learning systems*

| Book | Focus |
|------|-------|
| 📘 **Dive into Deep Learning (d2l.ai)** — Zhang et al. | Interactive, code-first deep learning with full math |
| 📗 **Deep Learning** — Goodfellow, Bengio & Courville | The theoretical bible of deep learning |
